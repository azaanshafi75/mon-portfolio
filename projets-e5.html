Situation 01

Configuration VLAN avec DHCP et routage inter-VLAN

Segmentation du réseau de l'entreprise ZNK en 4 VLANs (Administration, RH, Finance, IT) avec mise en place d'un routage "Router-on-a-Stick" et activation du DHCP sur le routeur pour automatiser l'adressage IP du parc.

Cisco Packet Tracer VLANs Router-on-a-Stick DHCP dot1Q
📄 Voir le PDF

Problématique

Réseau plat sans segmentation, gestion IP manuelle, sécurité insuffisante et congestions de trafic broadcast.

Solution mise en place

  • Création de 4 VLANs dédiés par service
  • Configuration du switch en mode trunk
  • Routage inter-VLAN via sous-interfaces (dot1Q)
  • Pools DHCP distincts par VLAN

Matériel utilisé

1 Routeur 2911, 1 Switch 2960, 4 PCs — Simulation Cisco Packet Tracer

Validation

Tests ICMP (ping) entre VLANs, vérification des baux DHCP attribués correctement.

Situation 02

Déploiement d'un réseau avec filtrage par ACL

Sécurisation du réseau de l'entreprise ZNK via des listes de contrôle d'accès (ACL) sur un routeur Cisco. Autorisation du trafic HTTP vers Serveur 1, blocage total vers Serveur 2 hébergeant des données sensibles.

Cisco ACL Sécurité réseau Routes statiques HTTP / Telnet
📄 Voir le PDF

Objectif

Autoriser l'accès HTTP entre PC1 et Serveur 1, tout en bloquant strictement toute communication vers Serveur 2.

Configuration

  • R1 : interfaces + routes statiques
  • R2 (pare-feu) : ACLs étendue sur Gi0/0
  • Permit HTTP port 80 vers 192.168.2.1
  • Deny tout accès vers 192.168.3.1

Tests de validation

  • Telnet port 80 → Serveur 1 : Succès
  • Telnet port 80 → Serveur 2 : Timeout
  • Navigateur : Serveur 1 accessible, Serveur 2 bloqué

Choix technique

ACLs Cisco retenues pour leur intégration native aux routeurs et leur contrôle granulaire du trafic.

pfSense Proxmox VE Pare-feu Virtualisation NAT

Architecture déployée

  • pfSense : WAN + LAN (192.168.10.254)

Situation 05

Application de GPO pour la sécurisation des postes clients

Mise en place de 4 Stratégies de Groupe (GPO) pour l'entreprise G-TECH : mappage automatique de lecteur réseau, fond d'écran imposé, blocage du panneau de configuration et désactivation de l'invite de commandes.

GPO Active Directory GPMC Windows Server Sécurité postes
📄 Voir le PDF

4 GPO déployées

  • GPO 1 : Mappage lecteur réseau (P: → \\SRV-ad-001\dossier)
  • GPO 2 : Fond d'écran personnalisé entreprise
  • GPO 3 : Blocage accès Panneau de configuration
  • GPO 4 : Désactivation invite de commandes (CMD)

Méthode

  • Création OU "UTILISATEUR RESTREINT"
  • Ajout utilisateur TESTCONFIG dans l'OU
  • Application des GPO via GPMC
  • Validation avec gpupdate /force

Pourquoi les GPO ?

Solution Microsoft native, sans coût additionnel, contrôle granulaire par OU. Standard de l'industrie pour la gestion centralisée.

Résultats

Lecteur réseau Z: mappé automatiquement, restrictions CMD et Panneau de config appliquées, fond d'écran uniformisé.

Situation 06

Accès distant SSH avec authentification par clés asymétriques

Sécurisation de l'administration à distance d'un serveur Debian. Remplacement de l'authentification par mot de passe par des clés RSA 4096 bits, durcissement SSH (changement de port, interdiction root) et pare-feu UFW.

SSH RSA 4096 bits Debian Linux UFW OpenSSH
📄 Voir le PDF

Étapes clés

  • Création utilisateur adminuser + groupe it-team
  • Installation OpenSSH-server
  • Génération clé RSA 4096 bits (PowerShell)
  • Transfert clé publique → authorized_keys

Durcissement SSH (/etc/ssh/sshd_config)

  • Port 2222
  • PermitRootLogin no
  • PubkeyAuthentication yes
  • PasswordAuthentication no

Pare-feu UFW

Ouverture du port 2222/tcp uniquement. Blocage de tous les autres accès non autorisés.

Tests de connexion

Connexion réussie depuis Debian (localhost) et depuis Windows : ssh -p 2222 adminuser@192.168.1.10

Situation 07

Redondance de passerelle via le protocole HSRP

Mise en place de la haute disponibilité réseau (HA) avec HSRP Cisco pour éliminer le point de défaillance unique (SPOF). Architecture active/active avec load balancing entre 2 routeurs sur 2 VLANs — continuité de service 24/7.

HSRP Haute disponibilité Load Balancing Failover Cisco
📄 Voir le PDF

Architecture HSRP

  • Groupe 10 (VLAN 10) : IP virtuelle 192.168.10.1
  • Groupe 20 (VLAN 20) : IP virtuelle 192.168.20.1
  • Router A : Chef VLAN 10 / Secours VLAN 20
  • Router B : Chef VLAN 20 / Secours VLAN 10

Tests de recette

  • Test A : show standby brief → Load Balancing actif ✓
  • Test B : Ping passerelle virtuelle depuis PC ✓
  • Test C : Simulation panne (failover) — convergence rapide ✓

Résultat du failover

Lors de la coupure d'un câble, seulement 1-2 requêtes ICMP perdues avant que le routeur en secours prenne le relais automatiquement.

Budget projet

2× Cisco ISR 4331 (~5 000€), 2× Switch Catalyst 2960-X (~1 600€), câblage (~600€). Total : 9 450€ HT
